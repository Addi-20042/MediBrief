# Weekly GitHub Workflow

Use this lightweight routine to keep your GitHub profile active in a real, professional way.

## Every Week

1. Pick one real improvement on one project.
2. Create a focused branch.
3. Make 1 to 3 small meaningful commits.
4. Update README or screenshots if the change affects presentation.
5. Open a clean pull request or merge with a clear summary.

## Good Weekly Tasks

- improve a README
- fix a bug
- add one polished feature
- clean up UI states
- improve validation or error handling
- write or improve tests
- add deployment notes

## What to Avoid

- pushing random incomplete code
- giant commits with unrelated changes
- fake activity commits
- leaving broken docs or dead links

## Monthly Check

- refresh pinned repositories
- archive weak or unfinished public repos
- improve one repo description
- add one screenshot or demo link to a flagship project
- update your profile README if your focus changes
